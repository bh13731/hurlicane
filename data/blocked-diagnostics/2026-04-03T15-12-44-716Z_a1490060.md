# Plan

## Goal
Fix Gmail API 429 "Too many concurrent requests for user" errors by throttling concurrent Gmail calls and eliminating Sentry noise from retryable rate-limit errors.

## Assessment

**Root cause**: two compounding issues caused the Sentry error.

### Issue 1: Premature + double Sentry reporting for rate-limit errors
`lib/gmail.py` previously captured every exception inside `with_backoff()`, including retryable 429s that later succeeded. Callers in `dashboard_services/analysis.py` already capture exceptions when `with_backoff` re-raises, so the correct fix was to remove Sentry capture from `with_backoff()` entirely.

### Issue 2: Excessive concurrent Gmail API calls
The analysis pipeline uses an outer pool across firms plus an inner per-firm pool for Gmail work. In the hot path this can create ~24-32 concurrent Gmail requests, which exceeds Gmail's per-user concurrency budget and triggers 429s.

### Affected files
- `lib/gmail.py` — Sentry removed from `with_backoff`; global semaphore added; imports `GMAIL_CONCURRENCY`.
